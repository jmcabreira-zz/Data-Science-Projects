Published on Sep 15, 2018

In this video we go over how to deploy software updates with SCCM and how to setup automatic deployment rules (ADR) so you can automate the process. I go over the steps to install the WSUS role on Windows Server as well as how to configure the SUP (Software update Point) role within SCCM. I will also show you step by step how to create Software Update Groups so you can use that as your group to deploy windows updates. Next we will go through Syncing with Microsoft update so you will be able to update your Updates Catalog. Once that is configured we go over setting up maintenance windows and restart times for the client settings so that the updates will install during the hours that you specify. The logs you would want to look for are the deploymentupdates.log, updatestore.log and the ruleegnine.log
